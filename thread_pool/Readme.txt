线程池的作用
http://www.cnblogs.com/zping/archive/2008/10/29/1322440.html
简单地说，反复操作线程的创建和销毁是很消耗CPU的事，如果在负载较大时仍然去做这样的工作就可能造成很多任务处理不及时或者更大危害。
因此一种解决办法就是提前创建好一些线程，将其休眠，用某种数据结构将其管理起来。
当任务来到时，分配其中一个线程为其工作，完成后将线程收回。这样也算做是用空间换取时间的办法。这就是线程池的原理。

线程池所考虑的主要问题：
1，创建多大数量的线程?
2，以何种数据结构来管理线程?
3, 如何调度线程？
4，线程创建后如何安置状态？任务完成后如何安置？
